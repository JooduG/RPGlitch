# RPGlitch Onboarding Guide

Welcome to **RPGlitch**! This guide is designed to help new developers understand the high-level architecture, the key components, and the design philosophy of the project.

## 1. Project Overview

**RPGlitch** is a next-generation AI Roleplay Engine built on the Perchance platform. It is a Local-First web application that turns your browser into a sophisticated RPG tabletop with a Simulation-Driven Architecture.

- **Languages:** JavaScript, Svelte, CSS, HTML, Markdown
- **Frameworks:** Svelte 5, Tailwind CSS, Vite, Dexie.js

## 2. Architecture Layers

The engine strictly adheres to a reactive flow (The Triad Protocol), separated into distinct sovereign modules:

1. **UI Layer:** Svelte 5 components (Atomic design: atoms, molecules, organisms).
   - _Dependencies:_ State Layer
2. **State Layer:** Svelte 5 Runes (`$state`, `$derived`, `$effect`) managing reactivity.
   - _Dependencies:_ Engine Layer, Data Layer
3. **Engine Layer:** Core simulation loop and time tracking (Rounds and Turns).
   - _Dependencies:_ Data Layer
4. **Intelligence Layer:** AI Kernel, context brokering, and prompt generation for entities.
   - _Dependencies:_ Platform Layer (Perchance/Local)
5. **Data Layer:** IndexedDB persistence via Dexie.js for memory and state recovery.
6. **Media Layer:** Audio and Visual aesthetics rendering (The Chalk Regime).

## 3. Key Concepts & The Red Thread

- **The Triad Protocol:**
  1. _The Spec:_ Deep lore and taxonomies.
  2. _The State:_ Reactive Svelte 5 Runes.
  3. _The Echo:_ Persistent logs (Dexie.js) providing context to the AI.
- **Svelte 5 Runes Only:** Do not use `export let`, `$:` or `writable()`. Rely exclusively on Runes.
- **The Chalk Regime:** The minimalist, Nordic-inspired visual identity.
- **Local-First & Perchance:** The engine compiles to a single HTML file (PWA) suitable for Perchance's iframe architecture.
- **Simulation Loop:** A macro-state (Round) containing micro-states (Turns) for AI Characters and the User Persona.

## 4. Guided Tour

When exploring the codebase for the first time, follow this path to understand the execution lifecycle:

1. **`src/main.js`**: The entry point of the application. It bootstraps the Svelte environment and loads the global systems.
2. **`src/engine/chrono.svelte.js`**: The temporal manager. This is where Rounds and Turns are orchestrated and where the simulation physics are enforced.
3. **`src/intelligence/kernel.js`**: The AI loop. It takes the engine's current state, builds a context prompt, and orchestrates AI responses.
4. **`src/data/db.js`**: The Dexie.js schema definition where you can see exactly how Characters, Fractals, and History are persisted locally.
