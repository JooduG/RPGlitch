## Utilities

### `guard-no-output-edits.js`

**Purpose:** Prevents accidental commits of edited files in `build/output/`.

**How it works:**
- Checks git status for modified files
- Exits with error if any `build/output/` files are changed
- Intended as pre-commit hook or CI check

**Usage:**
```bash
node tools/guard-no-output-edits.js
```

**Why this exists:**
Files in `build/output/` are generated by build scripts. Editing them directly is incorrect - you should edit source files in `apps/` instead.

**How to integrate as git hook:**
```bash
# Add to .git/hooks/pre-commit (create if doesn't exist):
#!/bin/bash
node tools/guard-no-output-edits.js || exit 1
```

### `diagnostics/automation-collect-diagnostics.js`

**Purpose:** Collects diagnostic information for troubleshooting issues.

**Usage:**
```bash
node tools/diagnostics/automation-collect-diagnostics.js
```

**What it collects:**
- System information
- Environment variables (filtered for security)
- Git status and branch information
- Package versions
- File structure analysis

**When to use:**
- Debugging build failures
- Troubleshooting environment issues
- Creating bug reports with full context

## Integration Points

### Git Hooks
The `guard-no-output-edits.js` script is designed to be used as a git pre-commit hook.

**To enable:**
```bash
echo '#!/bin/bash
node tools/guard-no-output-edits.js || exit 1' > .git/hooks/pre-commit
chmod +x .git/hooks/pre-commit
```

### CI/CD
Consider adding these checks to GitHub workflows:
- `guard-no-output-edits.js` - Prevent output file changes
- `diagnostics/automation-collect-diagnostics.js` - Collect context on failures
