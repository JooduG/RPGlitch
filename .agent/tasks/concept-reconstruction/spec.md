# Feature Spec: Concept Reconstruction (Meridian)

## Vision

Transform the fragmented `knowledge/lab` directory into a high-fidelity "Meridian" sequence. This ensures that every speculative idea or potential feature is logically categorized, easy to find, and aligned with the project's "Red Thread" and Skill Matrix.

## Objectives

- **Deconstruct**: Extract raw intelligence from 9 legacy files across 4 subdirectories.
- **Merge**: Eliminate redundant files (e.g., `ideas.md` vs `mechanics-concepts.md`).
- **Simplify**: Flatten the directory structure into a single `lab/` root.
- **Align**: Update terminology to reflect the **Skill Matrix** (Engine, UI, Polish, Data, Security).
- **Red Thread**: Ensure a logical flow from raw ideation to complex simulation logic.

## Target Structure (The Meridian Lab)

1. `lab/01-ideation.md`: High-level brainstorming and the "Maybe Pile."
2. `lab/02-interaction.md`: Sensory, immersion, and atmospheric concepts.
3. `lab/03-mechanics.md`: Speculative gameplay and physics engines.
4. `lab/04-logic.md`: Social mesh and psychological simulation models.
5. `lab/05-strategy.md`: Risk mitigation, premortems, and scaling vision.

## Acceptance Criteria

- [ ] No nested subdirectories remain in `knowledge/lab/`.
- [ ] 5 Meridian files created in `lab/` with consistent headers.
- [ ] `canon/` folder renamed to `atlas/`.
- [ ] Zero references to legacy "Pillars" (Gamemaster, Artificer, etc.) in concept docs.
- [ ] `config.yaml` updated to reflect new concept paths.
- [ ] Legacy files purged.
