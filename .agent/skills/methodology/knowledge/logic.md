# Cognitive & Operational Logic

This document defines the A-C-M-Q logical framework and the operational workflow patterns for the RPGlitch Core.

## 1. Cognitive Architecture (A-C-M-Q)

### Ambiguity Resolution (A-Score)

Before executing logic, evaluate the intent.

- **A1-A2**: Direct execution.
- **A3-A4**: Propose options.
- **A5**: Critical blockage.

### Complexity Routing (C-Level)

- **C1 (Reflex)**: Direct code edit.
- **C2 (Planning)**: Sequential thinking.
- **C3 (Metacognition)**: Self-audit.
- **C4-C6**: High-level frameworks.

---

## 2. Operational Workflow Patterns

### Sequential Workflows

For complex tasks, break operations into clear, verifiable steps.

Example:

1. **Analyze**: Identify the target state.
2. **Execute**: Perform the primary mutation.
3. **Verify**: Run the project audit.

---

## 3. The Meridian Execution (M-Sequence)

Apply the 5-step loop to all logical transitions:

1. **Input** -> 2. **Sanity** -> 3. **Execution** -> 4. **Persistence** -> 5. **Expression**.
