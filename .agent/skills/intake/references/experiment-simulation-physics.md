# The Physics Engine (Time & Permanence)

> Red Thread: The underlying logic governing risk, time flow, and continuous physical state.

## 1. Chrono Kinetics (Time Simulation)

Simulation speed scales inversely with content density.

- Micro (Seconds): Decelerate for dialogue, combat, or intimacy.
- Meso (Minutes): Real-time flow for specific, ongoing instructions.
- Macro (Hours/Days): Execute time jumps.
- **Prevent empty time jumps by requiring a Sensory Bridge (e.g., light shift, sound decay) before advancing the clock.**

## 2. The Entropy Engine

Every high-stakes action generates a chaos seed (1-100) determining visceral outcomes.

- 01-15: Critical Failure + New Threat.
- 16-40: Struggle / Resistance.
- 41-75: Competent Flow.
- 76-95: High Impact.
- 96-100: Permanent Change / Lethality.

## 3. Object Permanence & Entity Snapshots

Maintaining continuous state across the Simulation.

- Anchor System: Persistently track introduced objects, wounds, or environmental damage until explicitly resolved in the Simulation.
- Dual Realities: Maintain the static library state (`db.entities`) separately from the volatile Simulation state.
- **Generate a Growth Report upon Simulation conclusion, prompting the user to either merge the active data back to `db.entities` or revert to the initial snapshot.**
