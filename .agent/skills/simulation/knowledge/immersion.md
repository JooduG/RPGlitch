# 👁️ Diegetic Immersion Protocol

> **Skill:** simulation / mesmer
> **Red Thread:** UI elements exist WITHIN the fiction, treated as physical objects.

## 📟 1. Diegetic UI Types

| Type       | Name                 | Purpose                                   |
| :--------- | :------------------- | :---------------------------------------- |
| **Type A** | **The Viewport**     | Reading text (Books, terminals, letters). |
| **Type B** | **The Interactable** | Hidden checks (Chests, envelopes, doors). |
| **Type C** | **The Construct**    | Non-physical items (Magic, holograms).    |

## 📐 2. Implementation Rules

- **Inline Isolation:** Use inline styles where portability across sessions is required.
- **Narrative Anchoring:** UI must be described in prose before appearing.
- **Minimalist Feedback:** Atmosphere over "gamey" UI alerts.
- **Chalk Regime:** Adhere to defined color tokens and soft depth.
