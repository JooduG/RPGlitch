## 2. The Brain (A-C-Q Protocol)

Define the Clarity Gate constraints specific to this skill.

- **A-Score Requirements**: A4 (Critical). Reflection is literally the tool used to resolve A3/A4 ambiguity, and strategy is inherently open-ended.
- **C-Level Tools**: C4-C6 (Reframing, Decision, Science). Requires deep MCP reasoning tools.

## 3. Capabilities

- **Logical Arbitration**: Resolving conflicting code requirements.
- **Bug Eradication**: Applying the scientific method to impossible bugs.
- **Architectural & Product Design**: Trading off Option A vs B, and finding UX improvements with massive impact for low effort.
- **Feature Ideation**: Suggesting high-leverage capabilities based on the core app vision.

## 4. Procedures

1. **Analyze (Bugs/Logic)**: Break down the conflict into hypothesis -> test -> conclude using waldzell reasoning tools.
2. **Brainstorm (Product)**: Map current usage patterns -> Identify friction -> Propose elegant, high-ROI, low-code solutions.

## 5. Anti-Patterns

| Pattern                                     | Mitigation                                                                   |
|: ------------------------------------------ |: --------------------------------------------------------------------------- |
| **Skipping Clarity Gate for complex tasks** | Forbidden. All C2+ tasks require ambiguity assessment first.                 |
| **Guessing at A3+ ambiguity**               | Forbidden. Propose or present options; never assume intent.                  |
| **Proposing 6-month rewrites**              | Avoid. Execution strategy should focus on high ROI, low-friction improvements. |
