# 🧩 Skill: Sequential Thinking

> **The Engine:** Structured, step-by-step analysis for complex logic.

## 1. When to Use

- **Complexity:** L2 (Logic) or higher.
- **Trigger:** Multi-file changes, slight ambiguity, planning phases.
- **Goal:** Prevent "jump to solution" errors.

## 2. The Protocol

1.  **Call the Tool:** Use `mcp-sequentialthinking-tools` (specifically `sequentialthinking_tools` if available).
2.  **Steps:** Break the problem down into logical steps (1-N).
3.  **Output:** A clear plan of action before writing code.

## 3. Usage Pattern

```text
"Break this down. Steps 1-N..."
```
