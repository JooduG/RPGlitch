# Specification & Idea Workshop

> "Every chaotic vibe is distilled into a sharp, functional reality. Code without a spec is guessing. We define what we're building before we touch the engine."

## Overview

The `specification` skill is the RPGlitch Engine's filter for product thinking. It manages the journey from raw inspiration to technical blueprint, ensuring we build the _right_ thing before we worry about building it _right_.

### The Concept Lifecycle

1. **Phase 1: Diverge**: Interrogate ambiguous requests to find the core mechanic.
2. **Phase 2: Handshake**: Reflect intent back to the user for confirmation.
3. **Phase 3: Converge**: Evaluate directions against the "Nordic Aesthetic" and technical feasibility.
4. **Phase 4: Specification**: Draft the final blueprint in `SPEC.md` or a feature-specific doc.

## How It Works

### 1. Intent Validation

Never implement until the high-level vision is approved and assumptions are surfaced.

### 2. The Blueprint

Every feature spec must contain:

- **Objective**: What and why?
- **Success Criteria**: Testable conditions that prove completion.
- **Boundaries**: "Always do" and "Never do" rules.
- **Logic Path**: High-level data flow and state mutations.
