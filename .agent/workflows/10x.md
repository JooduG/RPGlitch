# [/10x](./10x.md) - Strategic Excellence

> **Goal:** Find 10x product opportunities and high-leverage improvements. Not 10% better—10x more valuable.

## 1. Triggers

- **Request**: "What should we build next?", "10x this", "product strategy".
- **Slash Command**: **[/10x](./10x.md)**

## 2. Context Injection

- **Rules**: [Foundation](../rules/01-foundation.md).
- **Rules**: [Intelligence](../rules/05-intelligence.md).
- **State**: [.agent/project-management/mission-board.md](../project-management/mission-board.md).

## 3. The 10x Mindset

- **Founder Mentality**: Think like you own the product.
- **Value over Features**: Focus on what makes the product indispensable.
- **Massive Opportunity**: Look for new markets or capabilities.
- **Medium Leverage**: Force multipliers on what already works.
- **Small Gems**: Tiny changes with outsized impact.

## 4. Procedure

### Phase 1: Current Value Audit
1.  **What problem does RPGlitch solve today?**
2.  **What's the core action users take?**
3.  **What's the single biggest friction point?**

### Phase 2: 10x Brainstorming
Think across three scales:
- **Massive**: Platform shifts (e.g., Multiplayer, AI-driven worldbuilding).
- **Medium**: Experience multipliers (e.g., Instant asset generation, Deep memory).
- **Small**: Utility gems (e.g., One-click character swap, Persistent state sync).

### Phase 3: Ruthless Evaluation
Score each idea from **1 (Low)** to **10 (High)** across:
- **Value**: Impact on the user.
- **Leverage**: Effort vs. Output ratio.
- **Defensibility**: Does it build a moat?

### Phase 4: Output (The Strategy Document)
All output must be written to:
`[.agent/project-management/strategy/session-YYYY-MM-DD.md]`

Wait for user approval on the strategy before moving to **[/01-plan](./01-plan.md)**.

## 5. Anti-Patterns

- **Incrementalism**: Suggesting safe, 10% improvements.
- **Feature Bloat**: Adding features for the sake of features.
- **Ambiguity**: Proposing "better UX" without a specific mechanic.
