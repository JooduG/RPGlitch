# Standard Task Workflow

## 1. Context & Setup

- **Rule**: Never start coding without understanding the "Why".
- **Action**: Read the `spec.md` and `plan.md` for the current track.

## 2. Test Driven Development (TDD)

- **Red**: Write a failing test for the functionality.
- **Green**: Implementation code to pass the test.
- **Refactor**: Optimize cleanliness and performance.

## 3. Implementation Steps

- **Atomic Commits**: Commit often with conventional messages (e.g., `feat:`, `fix:`, `refactor:`).
- **Verification**: Run tests and linters after every significant change.
- **User check**: If unsure about a UI change or logic decision, ask the user.

## 4. Definition of Done

- All tests pass.
- Linter is happy.
- Code is documented.
- No "TODO" comments left behind.
