# 🎯 Vision (The Red Thread)

> **Vision:** A local-first, genre-agnostic engine where state drives reality, and narrative is forged through recursive intelligence.
> **Red Thread:** Every mechanic serves the convergence of story and state.

## 1. The Core Philosophy

RPGlitch is a **Local-First Reactive Monolith**. We believe that roleplay tools should provide "Physics" for immersive storytelling, not just empty chat boxes.

### The Triad Protocol

We bridge creative prose and mechanical truth through three layers:

- **The Spec (Blueprint):** Deep lore and character archetypes.
- **The State (Live):** Reactive Runes mirroring psychological and physical reality.
- **The Echo (History):** Persistent logs providing weight to every decision.

## 2. Strategic Objectives

### 👁️ Diegetic Immersion

The UI is a canvas within the fiction. Information is atmospheric, using the "Chalk Regime" aesthetics to minimize noise and maximize sensory depth.

### 🧠 Recursive Intelligence

Intelligence is a logic pillar. The **Engine** orchestrates the user's input, the **Security** layer enforces laws of physics, and **Data** ensures the world remembers.

### 🛡️ Freedom & Privacy

A **Skill-Based Architecture** decouples logic from sensory output. The user owns their data via IndexedDB; there is no central server.

## 3. Product Integrity

1. **Local-First**: Persistence via `Dexie.js` is mandatory before network sync.
2. **Reactive**: Zero imperative UI state. Use Svelte 5 Runes.
3. **Aesthetic**: All visuals must adhere to the Design System tokens.
4. **Agentic**: The agent remains a collaborator, maintaining the blueprint as the single source of truth.
