# Product Guidelines

## Prose and Narrative Style

- **Dynamic Adaptability:** The narrative engine supports a range of styles, from evocative to technical, that can update based on the current story context or user preference.
- **Voice and Tone:** Clear, high-fidelity prose that prioritizes immersion while allowing the AI characters and the "fractal" (world/setting) to communicate with distinct voices.

## Visual Design (The Chalk Regime)

- **Minimalist Foundation:** Use a high-contrast, monochromatic base to ensure that "signature colors"—such as character-specific chat bubble backgrounds—can add personality without visual clutter.
- **Diegetic Consistency:** All UI elements should feel like part of the story world, adhering to the "Chalk Regime" design tokens.

## User Experience and Interaction

- **Intuitive Context:** Favor "meaningful interactions" over explicit UI controls. For example, clicking an empty element should trigger the logical next step (e.g., selecting a character) rather than requiring a separate "add" button.
- **Minimalist Restraint:** Show only the information and tools necessary for the current narrative moment, maintaining a focus on the story.

## Agentic and AI Interaction

- **Multi-Layered Communication:** Interaction occurs through AI characters, Out-of-Character (/OOC) system messages, and the "fractal" (world/setting) sending direct messages.
- **Supportive Scaffolding:** The AI "Director" provides a foundation for the procedural short stories, ensuring narrative coherence and character consistency.
