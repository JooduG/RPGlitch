# Build System Documentation

This directory contains the build pipeline for the monorepo, including scripts, configurations, vendored libraries, and build outputs.

## Directory Structure

```text
build/
├── scripts/           # Build automation scripts
├── local_libs/        # Vendored dependencies (Pico.css, Dexie, DOMPurify, etc.)
├── output/            # Generated artifacts (DO NOT EDIT)
└── README.md          # This file
```

## Scripts Overview

### Primary Build Scripts

#### `sync.js` (ES Module)
**Purpose:** Central orchestrator for syncing configurations, libraries, and generating artifacts.

**Supported Flags:**
- `--all` - Run all sync operations (default if no flags provided)
- `--libs` - Download and verify vendored libraries
- `--ignores` - Sync ignore patterns from `ignores.master.json`
- `--mcp` - Generate `mcp.json` from `mcp.master.json` with environment variable substitution
- `--hub` - Generate hub.md overview file

**NPM Scripts:**
```bash
npm run sync              # Run all sync operations
npm run sync:libs         # Sync libraries only
npm run sync:ignores      # Sync ignore patterns only
npm run sync:mcp          # Sync MCP configuration only
npm run sync:hub          # Generate hub file only
```

**Key Features:**
- SHA256 verification for all vendored libraries
- Atomic downloads (`.tmp` files)
- Environment variable substitution for MCP configs
- Centralized ignore pattern management

---

#### `build-app.js` (ES Module)
**Purpose:** Build individual Perchance applications (RPGlitch, ImageGlitch) into single inlined HTML files.

**Usage:**
```bash
node build/scripts/build-app.js <app-name>

# Or via NPM:
npm run build:rpglitch
npm run build:imageglitch
npm run build:apps        # Build all apps
```

**Process:**
1. Compiles SCSS → CSS (with Pico.css base + autoprefixer)
2. Bundles JavaScript modules with esbuild (ES6+ → IIFE, minified)
3. Inlines all CSS into `<style>` tags
4. Inlines all JavaScript into `<script type="module">` tags
5. Inlines vendored libraries from `/build/local_libs/`
6. Produces single HTML file in `/build/output/`

**Output:**
- `build/output/RPGlitch.html`
- `build/output/ImageGlitch.html`

---

#### `build-combine.cjs` (CommonJS)
**Purpose:** Combine repository files into Markdown overviews for fast review by humans or AI.

**Usage:**
```bash
npm run build:combine
```

**Output Files:**
- `combined-memory.md` - Memory bank archive files
- `combined-tests.md` - Test files
- `combined-tools.md` - Tool scripts
- `combined-readmes.md` - All README files
- `combined-everything.md` - Core documentation
- `combined-recent-changes.md` - Recent git commits

**Features:**
- Respects ignore patterns from `ignores.master.json`
- Filters by file extension (`.md`, `.js`, `.json`, etc.)
- Generates file trees and content summaries

## Vendored Libraries (`/build/local_libs/`)

All dependencies are vendored with SHA256 verification:

| Library | Version | Purpose |
|---------|---------|---------|
| `pico.min.css` | 2.0.6 | CSS framework base |
| `cash.min.js` | 8.1.5 | jQuery alternative (minimal DOM manipulation) |
| `dexie.js` | 4.0.7 | IndexedDB wrapper for state persistence |
| `purify.min.js` | 3.1.6 | DOMPurify for HTML sanitization |
| `_hyperscript.min.js` | 0.9.12 | Declarative UI interactions |

**Why Vendor?**
- Perchance single-file requirement (everything inlined)
- Offline development capability
- Version control and reproducibility
- SHA256 verification for security

**Updating Libraries:**
1. Update URL and SHA256 in `sync.js` SOURCES array
2. Run `npm run sync:libs`
3. Verify new checksum matches

**⚠️ DO NOT EDIT FILES IN THIS DIRECTORY**

All files are generated automatically by build scripts:

| File | Generated By | Purpose |
|------|--------------|---------|
| `RPGlitch.html` | `build-app.js` | RPGlitch application artifact |
| `ImageGlitch.html` | `build-app.js` | ImageGlitch application artifact |
| `hub.md` | `sync.js --hub` | Repository overview |
| `combined-*.md` | `build-combine.cjs` | Markdown file combinations |
| `.jest-result.json` | Jest (via npm test) | Test results for hub.md |
| `mcp.json` | `sync.js --mcp` | Generated MCP config (gitignored) |

## Common Workflows

### Initial Setup
```bash
npm ci                    # Install dependencies
npm run sync              # Sync configs and libs
npm run build:apps        # Build applications
npm test                  # Verify tests pass
```

### Development Workflow
```bash
# Make changes to app source files in apps/*/html, js, scss
npm run lint:fix          # Auto-fix linting issues
npm run build:rpglitch    # Rebuild specific app
npm test                  # Run tests
```

### Deployment Workflow
```bash
npm run deploy            # Sync → Lint Fix → Build → Test
```

## Troubleshooting

### "Checksum mismatch" error
- A vendored library failed verification
- Update the SHA256 hash in `sync.js` SOURCES array
- Or delete the file and re-run `npm run sync:libs`

### "Cannot find module" during build
- Run `npm ci` to reinstall dependencies
- Run `npm run sync:libs` to fetch vendored libraries

### Build produces empty HTML
- Check source files in `apps/*/html/`, `js/`, `scss/`
- Ensure Perchance left panel exists: `apps/*/[AppName]-left-panel.txt`
- Check for syntax errors in source JavaScript/SCSS

### Tests fail after build changes
- Babel config issue: Ensure `babel.config.cjs` uses `module.exports`
- Jest config: Ensure `jest.config.cjs` points to correct setup file

```
  ├─→ apps/*/scss/**/*.scss

  └─→ apps/*/*-left-panel.txt

build-combine.cjs
  ├─→ ignores.master.json
  └─→ (all repository files)
```

## Adding a New Build Script

1. **Create the script** in `build/scripts/`
   - Use `.js` for ES modules
   - Use `.cjs` for CommonJS (if using `require()`)

2. **Add to package.json**
   ```json
   "scripts": {
     "my-script": "node build/scripts/my-script.js"
   }
   ```

3. **Document it** in this README

4. **Test it** before committing
