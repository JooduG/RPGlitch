# TODO Template

## Structure

### Current Tasks

```markdown
### [Project/Area]
- [ ] Task with clear action
- [ ] Task with specific outcome
```

### Planned Tasks

```markdown
### [Phase/Category]
- [ ] Task with requirements
- [ ] Task with success criteria
```

### Ideas & Enhancements

```markdown
### [Category]
- Enhancement description
- Future feature consideration
```

### Technical Debt

```markdown
### [Area]
- Debt item with impact
- Quality improvement needed
```

### Blockers & Dependencies

```markdown
### Blockers
- Blocker with impact

### Dependencies
- Dependency with timeline
```

## Formatting

### Task Format

```markdown
- [ ] **Task Name**: Clear description with outcome
```

### Priority Indicators

- 🔥 **High Priority**
- ⚠️ **Medium Priority**
- 💡 **Low Priority**

### Status Indicators

- [ ] Not Started
- [🔄] In Progress
- [x] Completed (move to completed.md)

### Categories

- **Development**: Code, features, fixes
- **Documentation**: Guides, README, comments
- **Infrastructure**: Build, deploy, tools
- **Research**: Investigation, analysis
- **Planning**: Design, architecture

## Best Practices

1. Be specific with actionable descriptions
2. Define what "done" looks like
3. Mark priorities clearly
4. Move completed items to completed.md
5. Break down large tasks
6. Add time estimates when helpful
7. Note dependencies and blockers
