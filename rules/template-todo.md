# TODO TEMPLATE

> **TL;DR:** Template for organizing and tracking TODO items in the forward-looking planning system.

## 📋 **TODO STRUCTURE**

### **Current Tasks (In Progress)**

```markdown
### **[Project/Area Name]**
- [ ] Task description with clear action
- [ ] Another task with specific outcome
```

### **Planned Tasks (Ready to Start)**

```markdown
### **[Phase/Category Name]**
- [ ] Task with clear requirements
- [ ] Task with defined success criteria
```

### **Ideas & Future Enhancements**

```markdown
### **[Category Name]**
- Idea or enhancement description
- Future feature consideration
```

### **Technical Debt**

```markdown
### **[Area Name]**
- Technical debt item with impact description
- Code quality improvement needed
```

### **Blockers & Dependencies**

```markdown
### **Current Blockers**
- Blocker description with impact

### **Dependencies**
- Dependency description with timeline
```

## 📋 **FORMATTING GUIDELINES**

### **Task Format**

```markdown
- [ ] **Task Name**: Clear description with specific outcome
- [ ] **Another Task**: Description with success criteria
```

### **Priority Indicators**

- **High Priority**: Mark with 🔥 or **HIGH**
- **Medium Priority**: Mark with ⚠️ or **MEDIUM**  
- **Low Priority**: Mark with 💡 or **LOW**

### **Status Indicators**

- [ ] **Not Started**: Empty checkbox
- [🔄] **In Progress**: Progress indicator
- [x] **Completed**: Checked box (move to completed.md)

### **Categories**

- **Development**: Code, features, fixes
- **Documentation**: Guides, README, comments
- **Infrastructure**: Build, deploy, tools
- **Research**: Investigation, analysis
- **Planning**: Design, architecture

## 📋 **BEST PRACTICES**

1. **Be Specific**: Clear, actionable task descriptions
2. **Set Outcomes**: Define what "done" looks like
3. **Prioritize**: Mark high-priority items clearly
4. **Update Regularly**: Move completed items to completed.md
5. **Break Down**: Large tasks into smaller subtasks
6. **Time Estimates**: Add rough time estimates when helpful
7. **Dependencies**: Note what blocks or enables tasks
